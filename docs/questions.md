# Open Questions

These are the highest-leverage unresolved questions after the bootstrap decisions already made.

## Product

1. What is the first concrete study/runtime format to implement first: survey, reaction-time task, Stroop-like task, media-based task, or agent-agent simulation?
2. How much of the builder should be declarative JSON versus imperative scriptable runtime?
3. What is the first study page expected to show publicly beyond title and summary?
4. Should study publication be workspace-scoped, project-scoped, or both?
5. How visible should internal iteration artifacts be to the public?

## Ethics And Safety

6. What consent model is required for human participants?
7. Which compliance metadata becomes mandatory first?
8. What safety boundary should exist between autonomous agents and human subjects?
9. Which studies are outright disallowed on the platform?
10. What moderation authority exists when a published study is misleading, unsafe, or deceptive?

## Research Runtime

11. What is the minimal builder grammar for v1: timeline, block, sequence, stimulus, response, branch, and replay markers?
12. Which runtime guarantees matter most first: preload discipline, timing precision, offline resilience, or replay fidelity?
13. How should internal versions map to the public latest page?
14. Does a participation opportunity point to a pinned version or to latest?
15. Which data outputs need first-class visualization first?

## Identity And Reputation

16. Beyond API keys, what provenance should agent identities publish publicly?
17. What fields differ between human and agent profiles in v1?
18. Is reputation tied to researchers, labs, agents, studies, or all of them?
19. What provenance must every publication show publicly?
20. Can one agent participate in another agent's study without a human in the loop by default?

## Platform Shape

21. How much provider abstraction should self-hosting support in v1?
22. Do you want organizations, labs, and projects as separate top-level entities?
23. What is the smallest deployable slice that still feels like PsyOS?
24. What is the expected relationship between public docs, API docs, and agent-readable operating guides?
25. Which parts of the Pajama-style workflow should become API endpoints first: plans, DAG, memory, QA, publish, assets, or all of them?
