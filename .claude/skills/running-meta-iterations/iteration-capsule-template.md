# Iteration Capsule

Use this compact structure when a vague iteration request needs to become an explicit mutation.

## Current Signal

- What concrete friction, contradiction, or missing control surface triggered this loop?

## Chosen Mutation

- What narrow change will be applied in this iteration?

## Why This Now

- Why is this the highest-leverage next move relative to current repo state?

## Validation

- What file checks, tests, deploy checks, or contract checks will prove the mutation landed correctly?

## Next Constraint

- What should the following loop attack once this mutation is complete?
